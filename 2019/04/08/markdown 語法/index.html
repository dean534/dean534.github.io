markdown 語法

基本概念

markdown 是種輕量級的標記語言，最後轉換成 XML 格式呈現。
不同於同樣是XML格式的 HTML，markdown 注重語法「易讀易寫」，行內以標點符號標示格式，方便閱讀，非常適合用於撰寫文章或部落格。
由於最後會以轉換成 XML 檔，因此在行內除了 markdown 保留字元外，也可以使用 HTML 的語法撰寫。

特殊字元

在HTML中，無法直接呈現 < 與 & 這兩個具功能性的符號，需要使用實體字元（character entities）的方式撰寫。
而在 markdown 中會自行判別這些符號，並轉換成相對印的形式。舉例來說：

&copy;

會轉換成 © ，而

5 < 4

則會自動轉換成 5 &lt; 4 。

行級元素

段落與換行

在不同段落間可以使用空白的一行隔開。

在 markdown 中字句不會自動分行，若句子是接連不斷的，沒有空行的話，則會視為同一個段落，每個字句都不會分行。例如：

這是第一行
這是第二行

會編譯成

這是第一行
這是第二行

然而，你可以使用 <br /> 強制分行。

標籤（標題）

標籤可以分為兩種表現形式 Setext 與 Atx。

Setext形式是利用底線標示，利用 =（最高階標題）和 -（第二階標題）。

this is title
=============

this is another title
-------------

Atx形式則是在行首插入1到6個 # ，各對應到<h1>至<h6>。
通常建議會在句尾也加上 # 標記，方便閱讀。

# this is title #

## this is another title ##

以上兩者的字體大小式相同的。

區塊引言

在句首加入 >，可以使背景變為淡灰色，形成區塊引言，如下圖

this is block

區塊能夠巢狀或在內部加入其他 markdown 語法。

清單

無序

無序序列使用 - 、 * 與 + 於行頭標示，如：

*   listItem
*   listItem

+   listItem
+   listItem

-   listItem
-   listItem

均會被轉譯成：

  • listItem
  • listItem

有序

有序序列則使用數字。數字標題由第一個數字開始排序，與後面的數字無關。無論後面的數字為何都不會影響轉譯後生成的結果，如：

9.  listItem
5.  listItem
3.  listItem
16.  listItem

會轉變成

  1. listItem
  2. listItem
  3. listItem
  4. listItem

若要在一個序列中加入多段落，請使用tab或4個空白鍵縮排，如：

*   Lorem ipsum dolor sit amet, consectetuer adipiscing elit.
    Aliquam hendrerit mi posuere lectus. Vestibulum enim wisi,
    viverra nec, fringilla in, laoreet vitae, risus.

    Aliquam hendrerit mi posuere lectus. Vestibulum enim wisi,
    viverra nec, fringilla in, laoreet vitae, risus.

*   Donec sit amet nisl. Aliquam semper ipsum sit amet velit.
    Suspendisse id sem consectetuer libero luctus adipiscing.

會轉變為

  • Lorem ipsum dolor sit amet, consectetuer adipiscing elit.
    Aliquam hendrerit mi posuere lectus. Vestibulum enim wisi,
    viverra nec, fringilla in, laoreet vitae, risus.

    Aliquam hendrerit mi posuere lectus. Vestibulum enim wisi,
    viverra nec, fringilla in, laoreet vitae, risus.

  • Donec sit amet nisl. Aliquam semper ipsum sit amet velit.
    Suspendisse id sem consectetuer libero luctus adipiscing.

分隔線

可以使用連續三個或是以上的 * 、 - 與 _ 標示分隔線，此行不可以有其他的字元。

行內元素

強調

利用 * 與 _ 強調字句，若只有一個符號則會轉換成<em>（斜體），兩個則會轉換成 <strong>（粗體），如：

*single asterisks*

_single underscores_

**double asterisks**

__double underscores__

轉換成

single asterisks

single underscores

double asterisks

double underscores

行內程式碼

可以利用 ` ，包覆行內程式碼，經過編譯後轉換成 <code>。
若要在程式碼內顯示 ` ，則需要使用雙反影號，即

`` ` ``

連結

最簡易的連結使用 [顯示字句](超連結) 的語法。

若要在連結加入參考字句，則需要使用定義變數的方式，在句中加入 [顯示字句][參考變數]，
並在文章中任一位置定義變數 [參考變數]: 超連結 "參考字句"，舉例來說：

這是[google][google]連結
[google]: http://google.com.tw/ "go to google"

會變成「這是google連結」。

若【變數參數】與【顯示字句】相同，則可以簡寫成[顯示字句][]。

此外，markdown也支援使用 <> 的形式標記連結或是電子郵件的連結，如：

<address@example.com>
<http://google.com.tw/>

address@example.com
http://google.com.tw/

圖片

與連結相同可以使用行內或是變數的方式插入圖片，如：

![Alt text](/path/to/img.jpg)

![Alt text](/path/to/img.jpg "Optional title")

或是

![Alt text][id]

[id]: url/to/image  "Optional title attribute"

跳脫字元

markdown使用 \ 來顯示保留字元的符號。如：

\   反斜線
`   反引號
*   星號
_   底線
{}  大括號
[]  方括號
()  括號
#   井字號
+   加號
-   減號
.   英文句點
!   驚嘆號
